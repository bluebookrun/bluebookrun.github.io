http_test

Configure HTTP test case.

HTTP requests are grouped into test cases. Test cases can be executed with Bluebook CLI.

Each test case can also have one or more requests, where each request will be exeucted in the order they are listed.

Example

resource "http_test" "my_test_case" {
    steps = [
        ${http_step.my_step1.id},
        ${http_step.my_step2.id},
    ]
}

resource "http_step" "my_step1" {
    ...
}

resource "http_step" "my_step2" {
    ...
}

Inputs

  • steps — A list of http_step IDs to execute. The steps will be executed in the order they are listed.

Outputs

  • id - resource ID.
